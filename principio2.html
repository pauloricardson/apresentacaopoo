Princípio Aberto/Fechado - OCP

O princípio Aberto-Fechado é demonstrado na hierarquia de UnidadeSaude, onde a classe base está FECHADA para modificação (não alteramos seu código quando criamos novos tipos de unidades), mas ABERTA para extensão (podemos criar HospitalPublico, Ubs, PostoVolante e futuras unidades como DriveThruVacina ou UnidadeRural sem modificar o sistema existente). Cada nova unidade adiciona comportamentos específicos através da sobrescrita do método realizarAtendimento(), mantendo a compatibilidade com todo código que depende da abstração UnidadeSaude.
